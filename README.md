# EZ记账

**EZ记账**是一款面向自托管 [ezBookkeeping](https://github.com/mayswind/ezbookkeeping) 的轻量 Android 客户端。应用使用沉浸式 WebView 承载原始移动端界面，不在记账页面额外叠加标题栏或工具栏。

> 本项目是非官方客户端，不包含 ezBookkeeping 服务端，也不隶属于 ezBookkeeping 官方项目。

## 主要功能

- 全屏显示 ezBookkeeping 原始界面
- 同时配置本地地址和公网地址
- 有可复用线路时立即进入主页面，并在后台静默并行测速
- 本地与公网线路并行测速
- 自动、本地固定和公网固定三种线路模式
- 自动切换具备连续失败确认、性能阈值和60秒冷却期
- 原生错误恢复页面，可重新连接、测速、切换线路或修改地址
- 保持 Cookie 和登录状态
- 快捷中心手动刷新当前页面和网页返回历史
- 统一处理网页、首页、设置页和错误页返回操作
- 双指快速双击打开隐藏功能菜单
- 桌面长按快捷进入线路设置、安全设置、立即锁定和检查更新
- 指纹/面容、四位数字密码和九宫格图形锁
- 可配置后台锁定时间和熄屏立即锁定
- PIN和图形连续输错后启用递增等待限制
- 跟随系统深色模式并适配状态栏和导航栏
- 支持相机拍照、照片多选和普通附件上传
- 文件下载到 Android“下载/EZ记账”目录，完成后可直接打开
- 应用内检查 GitHub Releases 更新
- 严格阻止无效 HTTPS 证书和 HTTPS 页面中的 HTTP 混合内容

## v1.5.0 稳定内核

v1.5.0 重点重构应用内部结构，而不是继续堆叠界面功能：

- `MainActivity` 只负责协调页面和功能模块
- `AppStateMachine` 管理锁定、初始化、线路检测、网页加载、就绪、错误和设置状态
- `AppLifecycleCoordinator` 管理冷启动、后台返回、熄屏和外部设置页面
- `RouteCoordinator` 管理并行测速、自动切换、防抖冷却和手动线路模式
- `WebViewController` 管理 WebView、安全导航、上传和双指手势
- `DownloadController` 管理下载、权限和完成后打开文件
- `ServerSettingsPage` 和 `ErrorRecoveryPage` 提供独立原生页面
- 全面迁移到 Android Activity Result API
- 增加冷启动、后台超时、熄屏、旋转、取消验证、错误恢复和线路策略测试

## v1.5.6 返回逻辑统一

- 新增 `BackNavigationPolicy`，将返回决策从界面代码中独立出来
- 网页存在历史记录时优先返回上一页
- 网页没有历史记录但当前不是首页时，回到当前线路首页
- 从服务器设置页返回时恢复原有应用内容
- 从错误恢复页返回时优先重新建立已配置线路
- 真正位于根页面时才启用“再按一次退出”
- 用户继续触摸或正常导航后自动清除退出确认状态
- 使用 `OnBackPressedDispatcher` 统一三键导航、手势返回和预测性返回入口
- 桌面快捷方式统一进入返回增强 Activity

## v1.5.5 刷新方式调整

## v1.5.2 快速启动

- 自动模式优先复用上次成功线路，不再每次停留在检测页面
- 固定本地或固定公网模式直接进入对应线路
- 只配置一条线路时直接进入
- WebView开始加载后再进行后台静默测速
- 后台测速不覆盖当前页面，也不因轻微延迟差异立即切换
- 当前线路确实失败后，继续使用连续失败确认和备用线路回退
- 首次配置、地址变更或没有有效历史线路时，仍执行前台检测

## 线路模式

在记账页面双指快速双击，选择 **手动切换线路**：

- **自动选择**：并行检测两条线路，根据可用性和延迟选择，并使用防抖策略避免频繁跳换。
- **固定本地线路**：只使用局域网地址；不可用时进入原生恢复页，不自动切到公网。
- **固定公网线路**：只使用公网地址；不可用时进入原生恢复页，不自动切到本地。

自动模式的切换规则：

- 当前线路连续失败两次后才切换备用线路
- 自动切换后60秒内不因性能差异切回
- 两条线路都可用时，候选线路必须明显更快才会触发性能切换
- 手动测速只刷新检测结果，不强制改变当前线路模式

## 安全验证

在隐藏功能菜单中选择 **进入 App 的安全验证**：

- **指纹或面容**：使用 Android 系统 `BiometricPrompt`，并支持手机锁屏密码作为备用验证。
- **四位数字密码**：使用随机盐值和 PBKDF2 哈希保存在应用私有目录，不保存明文。
- **九宫格图形锁**：至少连接四个点，同样只保存加盐哈希。
- **自动锁定时间**：立即、15秒、1分钟、5分钟或仅完全退出后。
- **熄屏后立即锁定**：可单独开启或关闭。

已启用安全验证时，修改或关闭安全方式前需要再次验证当前方式。

## 首次配置

可填写一个或两个地址：

- **本地地址**：例如 `http://192.168.1.100:8080`
- **公网地址**：例如 `https://money.example.com`

只填写一个地址也可以正常使用。公网访问建议使用有效 HTTPS 证书。

## 自动构建

仓库包含两条正式 Android 工作流：

- **Android CI**：在提交和 PR 时运行单元测试并构建 Debug APK。
- **Signed Android Release**：通过手动运行或推送 `v*` 标签，使用固定签名证书构建 Release APK。

本地调试构建需要 Java 17、Android SDK 35 和 Gradle 8.9：

```bash
gradle --no-daemon :app:testDebugUnitTest :app:assembleDebug
```

## 固定签名

Release 构建强制使用固定签名。缺少签名变量时，Gradle 会中止 Release 构建，避免误发布不同证书签名的 APK。

固定签名证书 SHA-256：

```text
D2:9D:3F:58:1D:1A:90:B1:2A:06:73:17:C6:F7:29:57:83:72:7E:15:73:E7:C2:83:E0:9E:F3:98:CA:64:B7:99
```

GitHub Secrets 配置和正式构建方式见 [SIGNING_SETUP.md](SIGNING_SETUP.md)。

## 隐私

服务器地址、安全方式及密码/图形哈希只保存在手机应用私有目录。账号和账目数据直接在手机与用户配置的 ezBookkeeping 服务端之间传输，本客户端不提供中转服务。
